--------------------------------------------------------------------------------
1. WHAT WE ARE BUILDING (SCOPE)
--------------------------------------------------------------------------------

We are building a MINIMUM VIABLE version of the thesis:

  Frozen ResNet18 (ImageNet pretrained)
        |
        |  (features: 512-dim)
        v
  Bottleneck Adapter (trainable, ~16K params)
        |
        v
  Linear Head + ReLU  -->  Evidence e_k for each class
        |
        v
  Dirichlet alpha_k = e_k + 1
        |
        v
  Evidential Loss (MSE version + KL regularization)

We train this on a 5-way 5-shot task sampled from CIFAR-FS.
We evaluate on:
  - Accuracy (on 15-query images per class)
  - ECE (Expected Calibration Error)
  - Brier Score
  - OOD AUROC (in-distribution: CIFAR-FS, out-of-distribution: SVHN)

We compare against a BASELINE: same model but with Softmax + Cross Entropy.

--------------------------------------------------------------------------------
3. RESEARCH QUESTIONS ADDRESSED BY THIS DEMO
--------------------------------------------------------------------------------

From the original proposal, this demo directly addresses:
  - RQ2 (PARTIALLY): "Can Evidential Dirichlet Networks provide superior
                     calibration compared to Softmax when fine-tuning with
                     <500 trainable parameters?"
                     --> YES, we show this with a direct Softmax vs Evidential
                         comparison on CIFAR-FS.
  - RQ3 (PARTIALLY): "Does Bayesian loss improve OOD detection in low-data
                     regimes?"
                     --> YES, we show this using vacuity (u = K/S) as OOD score
                         on CIFAR-FS (in-dist) vs SVHN (OOD).

RQ1 and RQ4 are future work (mentioned in slides).

6. KEY DESIGN DECISIONS (why we chose these)
--------------------------------------------------------------------------------

Decision 1: Use ResNet18 + plain 224x224 images, not small-image variant.
  WHY: torchvision ResNet18 pretrained weights expect 224x224. CIFAR is 32x32,
       so we UPSAMPLE to 224 via torchvision transforms. Simpler than
       retraining a small-image backbone.

Decision 2: Use evidential MSE loss, not evidential cross-entropy.
  WHY: The Sensoy 2018 paper shows MSE version is most stable and gives best
       calibration. Reference: dougbrion/pytorch-classification-uncertainty.

Decision 3: Insert adapter ONLY after avgpool (between pool and classifier).
  WHY: Inserting inside residual blocks is tricky and error-prone in 2 days.
       A single adapter on 512-dim features is clean and enough to prove the
       concept.

Decision 4: Use ONE episode (not full episodic training loop).
  WHY: Full episodic meta-training takes hours and risks failure. One 5-way
       5-shot episode trained for 200 steps is enough to show the idea works.
       Multiple episodes can be added later; for pre-defence, one is enough.

Decision 5: SVHN for OOD (not TinyImageNet).
  WHY: SVHN is small (~60MB), auto-downloads via torchvision, and is
       "far-OOD" which gives the cleanest demo results. Near-OOD is harder
       and not needed for pre-defence.

Decision 6: Borrow evidential loss code from dougbrion/pytorch-classification-uncertainty.
  WHY: Cleanest and most-tested PyTorch implementation of Sensoy 2018.
       We will cite it in the slides.
